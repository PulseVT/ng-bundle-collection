Description

Private methods of ng-bundle-collection.Collection

Methods

  • __addOne(item, params)

    Adds single item to collection.

    To add an item, please use collection.add or collection.add_withToCache

    Parameters
    ParamTypeDetails
    itemobject

    Item to be added.

    paramsobject

    Params object of request which responded with this item.

    Returns
    object

    Added item

  • __calcParamsMark(params)

    Calculates string mark for parameters object

    Params mark is used for:

    • marking responses for requests and determining if the response is already cached
    • marking promises and determining if the request is already pending
    Parameters
    ParamTypeDetails
    paramsobject

    Params object

    Returns
    string

    The string mark of params

  • __callExtendFns(fns_arr)

    Invokes each extending function from array with passed params. Extending functions array is taken from Collection.extendFns

    Parameters
    ParamTypeDetails
    fns_arrarray

    Array of functions to be called

  • __callInterceptors(fns_arr, response, params)

    Invokes each interceptor from array (chain) with passed params. Extending functions array is taken from Collection.extendFns

    Parameters
    ParamTypeDetails
    fns_arrarray

    Array of functions to be called

    responseobjectarray

    Response from previous interceptor or otiginal response

    paramsobject

    Params object with which was the request made

    Returns
    object|array

    Value which is returned from the last interceptor in chain

  • __determineResponse(response, params)

    Formats the response to fetch action from backend.

    The format of resulting response is:

    - if the response is expected to be an Array, then the response is 
      		{
  			results: [
  				//array of items of array
  			],
  		}
    - if the response is expected to be an Object, then response is just the same as backend sent

    Adds __payload with params if the collection.config.respondWithPayload is true

    Each item comes decorated with collection.config.model

    Parameters
    ParamTypeDetails
    responseobject

    Success response from backend

    paramsobject

    Params object, with which was the request done.

    Returns
    object

    Formatted response

  • __error(response, params)

    Error callback for fetching action.

    • Calls extending error (collection.extendFns.fetch.e) functions
    • Calls extending final (collection.extendFns.fetch.f) functions
    Parameters
    ParamTypeDetails
    responseobject

    Error response from backend

    paramsobject

    Params object, with which was the request done.

  • __initCacheClearing()

    Adds extending function for fetch success (collection.extendFns.fetch.s), which clears promises from cache

    Invoked if the cache is disabled by config.withCaching === false

  • __initCaching()

    Enable caching by adding extending functions for fetch success (collection.extendFns.fetch.s) and for fetch error (collection.extendFns.fetch.e)

    Invoked if the cache is ensabled by config.withCaching === true

  • __invalidateParams(params)

    Deletes cached responses where the key corresponds to params object, which contains passed params

    Is called by Collection.invalidate for each params object

    Parameters
    ParamTypeDetails
    paramsobject

    The params object, which determines deleted cached responses

    Example

    This deletes all cache records, whose keys correspond to params object with

    'page_size' == 10 AND 'name' == 'asd'

      		collection.invalidate({page_size:10, name: 'asd'});
